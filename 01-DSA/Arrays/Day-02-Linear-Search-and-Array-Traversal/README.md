# Linear Search

1. What is Linear Search?

Linear search is a search where we compare each element with the value we try to find.

2. Why is it called Linear Search?

it's because we search it by checking each element.

3. What is the Best Case Time Complexity of Linear Search?

O(1)

4. What is the Worst Case Time Complexity of Linear Search?

O(n)

5. What is the Space Complexity of Linear Search?

O(1)

6. When should we use Linear Search?

Use Linear Search when the data is unsorted, the dataset is small, or you only need to perform an occasional search. 


7. Advantages of Linear Search

Works on sorted and unsorted arrays.
Simple to implement.
No extra space required.

8. Disadvantages of Linear Search

Slow for large datasets because it may need to check every element.

9. Real-life example of Linear Search

Searching for a name in an unsorted contact list or attendance sheet is a real-life example of Linear Search.

10. Difference between Linear Search and Binary Search? (Just your understanding. We'll study Binary Search later.)

Linear search works on all arrays but Binary search needs sorted arrays
